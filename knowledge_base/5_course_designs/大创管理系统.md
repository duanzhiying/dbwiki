## 项目概述

大学生创新创业训练计划（简称"大创"）是高校培养学生创新实践能力的重要平台。然而，传统的大创项目管理往往依赖纸质文件和线下流程，存在信息不透明、申报流程繁琐、数据统计困难等问题。

本系统基于 **Flask + MySQL** 技术栈，开发了一套面向学生和管理员双角色的 Web 大创项目管理系统，实现了项目申报、审核、查询和统计的全流程数字化管理。

**项目规模：**
- 后端代码：约 700 行 Python（Flask）
- 前端代码：约 3 个 HTML 页面（学生端、管理员端、登录页）
- 数据库：9 张关系表 + 1 个视图 + 1 个触发器
- 技术栈：Flask 2.0 / MySQL 8.0 / HTML5 + ES6+ / PyMySQL

## 需求分析

### 功能需求

系统分为 **学生** 和 **管理员** 两大角色，各自拥有独立的操作界面和权限。

#### 学生端功能

| 功能模块 | 功能描述 |

| 登录认证 | 输入学号选择学生角色登录，系统验证学号是否存在 |

| 公告通知 | 查看管理员发布的最新公告（通知/公告/紧急分类） |

#### 管理员端功能

| 功能模块 | 功能描述 |
|----------|----------|
| 统计概览 | 仪表盘展示学生总数、教师总数、项目数、经费总额、待审核报告数等 |

### 非功能需求

| 需求类型 | 具体指标 |
|----------|----------|
| 并发支持 | 支持 1000+ 用户同时使用 |
| 响应时间 | 系统响应时间 < 2 秒 |

## 数据库设计

### E-R 图设计

系统包含 **7 个核心实体** 和 **2 个从属实体**：

**核心实体：**
- 学生（students）
- 指导教师（teachers）
- 省级/国家级项目（prov_projects）
- 校级项目（uni_projects）
- 项目成员（project_members）
- 管理员（admins）
- 公告（announcements）

**从属实体：**
- 申报书（project_reports）
- 审核记录（audit_records）

**实体间关系：**
- 学生与项目之间为 **多对多** 关系，通过 `project_members` 关联表实现
- 教师与项目之间为 **一对多** 指导关系（一个项目有 1-2 名指导教师）
- 管理员与公告之间为 **一对多** 发布关系
- 申报书与审核记录之间为 **一对多** 关系

```sql
CREATE TABLE uni_projects (

CREATE TABLE IF NOT EXISTS project_reports (
```

#### 高级数据库对象

**视图：按专业统计在研项目数**

```sql
CREATE VIEW v_major_project_count AS
SELECT 
    s.major AS 专业名称,
    COUNT(DISTINCT pm.pno) AS 在研项目数
FROM students s
JOIN project_members pm ON s.sno = pm.sno
LEFT JOIN prov_projects pp ON pm.pno = pp.pno AND pp.status = '在研'
LEFT JOIN uni_projects up ON pm.pno = up.pno AND up.status = '在研'
WHERE pm.status = '有效'
  AND (pp.pno IS NOT NULL OR up.pno IS NOT NULL)
GROUP BY s.major;
```

**触发器：项目成员状态变更自动记录日志**

```sql
CREATE TABLE IF NOT EXISTS project_member_log (
    log_id INT AUTO_INCREMENT PRIMARY KEY COMMENT '日志ID',
    pno VARCHAR(15) NOT NULL COMMENT '项目编号',
    sno VARCHAR(10) NOT NULL COMMENT '学号',
    old_status ENUM('有效','失效') NOT NULL COMMENT '变更前状态',
    new_status ENUM('有效','失效') NOT NULL COMMENT '变更后状态',
    change_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '变更时间'
) COMMENT '项目成员状态变更日志';

DELIMITER //
CREATE TRIGGER trg_project_member_status
AFTER UPDATE ON project_members
FOR EACH ROW
BEGIN
    IF OLD.status != NEW.status THEN
        INSERT INTO project_member_log (pno, sno, old_status, new_status)
        VALUES (OLD.pno, OLD.sno, OLD.status, NEW.status);
    END IF;
END //
DELIMITER ;
```

```python
@app.route('/api/login', methods=['POST'])
def login():
    data = request.json
    sno = data.get('sno')
    get_db_connection()
    cursor = db.cursor()
    cursor.execute('SELECT * FROM students WHERE sno = %s', (sno,))
    student = cursor.fetchone()
    if student:
        columns = [desc[0] for desc in cursor.description]
        student_dict = dict(zip(columns, student))
        return jsonify({'success': True, 'student': student_dict})
    return jsonify({'success': False, 'message': '学号不存在'})
```

### 3. 我的项目与项目详情查看

**功能描述：** 学生登录后主页自动加载个人参与的项目列表，通过 GET `/api/student/projects/{sno}` 查询。点击项目可弹出详情弹窗展示项目信息、指导教师列表和项目成员列表。

**界面截图：**

**涉及文件：** `dashboard.html`、`app.py`（路由 `/api/student/projects/<sno>`、`/api/project/detail`）

**核心代码（后端）：**

```python
@app.route('/api/student/projects/<sno>', methods=['GET'])
def get_student_projects(sno):
    get_db_connection()
    cursor = db.cursor()
    cursor.execute('''
        SELECT pm.pno, pm.plevel, pm.role, pm.join_date, pm.leave_date, pm.status,
               pp.pname AS project_name, pp.level AS project_level, 
               pp.type, pp.year, pp.status AS project_status,
               up.pname AS uni_project_name, up.level AS uni_level, 
               up.type AS uni_type, up.year AS uni_year, up.status AS uni_status, up.final_grade
        FROM project_members pm
        LEFT JOIN prov_projects pp ON pm.pno = pp.pno AND pm.plevel = 'prov'
        LEFT JOIN uni_projects up ON pm.pno = up.pno AND pm.plevel = 'uni'
        WHERE pm.sno = %s AND pm.status = '有效'
    ''', (sno,))
    projects = cursor.fetchall()
    # 合并 prov 和 uni 的项目字段
    result = []
    for project in projects:
        project_dict = dict(zip(columns, project))
        project_dict['project_name'] = project_dict['project_name'] or project_dict['uni_project_name']
        project_dict['project_level'] = project_dict['project_level'] or project_dict['uni_level']
        # ...
        result.append(project_dict)
    return jsonify({'success': True, 'projects': result})
```

### 4. 申报书审核模块

**功能描述：** 管理员在"报告审核"标签页查看所有学生提交的申报书，可按审核状态（全部/待审核/已通过/已拒绝）筛选。对待审核的申报书可点击"通过/拒绝"按钮进行审核，审核时需输入审核意见（拒绝必填）。操作同时更新 `project_reports` 表状态并写入 `audit_records` 表实现审核留痕。已审核通过的申报书支持下载。

**界面截图：**

![alt text](course_designs/round4_1.png)

**涉及文件：** `admin-dashboard.html`、`app.py`（路由 `/api/admin/reports`、`/api/admin/audit-report`、`/api/admin/download-report/<id>`）

**核心代码（后端）：**

```python
@app.route('/api/admin/audit-report', methods=['POST'])
def audit_report():
    data = request.json
    report_id = data.get('report_id')
    admin_id = data.get('admin_id')
    audit_status = data.get('audit_status')
    audit_opinion = data.get('audit_opinion')
    
    get_db_connection()
    cursor = db.cursor()
    
    new_status = 'approved' if audit_status == '通过' else 'rejected'
    
    # 更新申报书状态
    cursor.execute('UPDATE project_reports SET status = %s WHERE id = %s', (new_status, report_id))
    
    # 写入审核记录
    cursor.execute('INSERT INTO audit_records (report_id, admin_id, audit_opinion, audit_status) VALUES (%s, %s, %s, %s)', 
                   (report_id, admin_id, audit_opinion, audit_status))
    
    db.commit()
    return jsonify({'success': True, 'message': '审核完成'})
```

```python
from dbutils.pooled_db import PooledDB

pool = PooledDB(

    host='localhost',
    port=3306,
    user='root',
    password='123456',
    database='data_demo'

def get_db_connection():

```

#### 2. 查询优化

- **合并多次 COUNT 为单次查询**：统计概览中的 9 次独立 COUNT 可考虑使用子查询合并为一条 SQL
- **使用 LIMIT 限制结果集**：公告查询已使用 `LIMIT 10`，其他列表查询应考虑分页
- **避免 SELECT \***：当前多处使用 `SELECT *`，建议明确指定所需字段

```sql
-- 合并统计查询示例
SELECT 
    (SELECT COUNT(*) FROM students) AS total_students,
```

- 引入 CSS 文件和 JavaScript 文件分离，减少 HTML 内联代码量
- 使用模板引擎（Jinja2）替代前端直接拼接 HTML 字符串

## 项目总结

### 技术收获

1. **Flask 后端开发流程**：掌握了 Flask 路由设计、请求参数解析、JSON 响应构建、CORS 跨域配置等核心技能。通过 RESTful API 设计完成了学生和管理员两套接口体系。

2. **MySQL 数据库设计实践**：从 E-R 图分析到关系表转换，深刻理解了实体关系映射、外键约束、枚举类型、视图和触发器的实际应用场景。特别在项目成员与项目的多对多关系设计中，通过关联表（project_members）实现了灵活的双向查询。

3. **前后端分离开发**：采用 Flask 提供 API + 原生 HTML 前端调用的模式，理解了 JSON 数据交互、localStorage 状态管理和异步请求处理。

4. **数据完整性保障**：通过外键约束保证引用完整性，使用触发器实现自动化日志记录，利用事务保证审核操作的原子性。

### 项目亮点

1. **双角色分权管理**：系统区分学生和管理员两种角色，各自拥有独立的操作界面和功能权限，学生端侧重查看和申报，管理员端侧重审核和配置。

2. **两级项目模型**：同时支持省级/国家级项目和校级项目两类大创项目，各自拥有独立的字段和状态枚举（如校级项目特有 `final_grade` 结项等级字段），并有各自完整的查询、统计和审核流程。

3. **完整审核留痕**：申报书审核不仅更新状态，还通过独立的审核记录表（audit_records）记录每一次审核操作（审核人、审核意见、审核时间、审核结果），便于事后追溯和审计。

4. **数据库完整性保证**：使用外键约束、枚举类型、视图和触发器，确保数据的一致性和完整性。触发器能够在项目成员状态变更时自动记录变更日志，便于历史追溯。

5. **动态 SQL 拼接**：项目查询模块支持多条件组合筛选，后端根据前端传入的参数动态拼接 WHERE 条件，兼顾查询灵活性和安全性（参数化查询防 SQL 注入）。

6. **级联数据清理**：删除学生时自动清理其项目成员关系和申报书记录，删除教师时自动将项目中关联的教师置空，避免产生孤立数据。
